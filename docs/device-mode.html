Device Mode & Mobile Emulation

As more and more users consume the web on mobile, the importance of responsive, mobile-friendly design grows. Web content needs to look and feel great across a wide variety of devices and network conditions. But ensuring high quality mobile experiences increases the complexity of testing and can lengthen development cycles.

Device mode brings the uniqueness of mobile testing to your browser tab through the power of mobile emulation.

  • Test responsive designs by emulating different screen sizes and resolutions, including retina displays.
  • Evaluate your site's performance on various networks, such as Edge and 3G, without affecting traffic to other tabs.
  • Accurately simulate device input for touch events, geolocation, and device orientation.
  • Enhance your current debugging workflow by combining device mode with the existing DevTools.

Note: Some of our documentation may be ahead of the stable version of Chrome. If you are unable to locate a feature, we recommend trying Chrome Canary which contains the latest version of the DevTools.

Enabling Device Mode

Turn on device mode by pressing the Toggle device mode icon . When device mode is enabled, the icon turns blue and the viewport transforms into a device emulator.

You can also toggle device mode on and off using the keyboard shortcut:
Ctrl+Shift+M (or Cmd+Shift+M on Mac).

Using the Screen Emulator

Device mode's screen emulator allows you to test the responsiveness of your site on various screen sizes, without the hassle of switching between multiple devices.

Get started with device presets

To jumpstart your debugging process, device mode is equipped with a variety of emulation presets. Quickly emulate a particular device by selecting a model from the preset dropdown.

These presets automatically configure device emulation in the following ways:

  • Specify the UA string for requests.
  • Set the device resolution and pixel ratio.
  • Enable touch emulation (if applicable).
  • Emulate mobile scrollbar overlays and meta viewport.
  • Autosize (boost) text for pages without a defined viewport.

Tips: Quickly alternate between portrait and landscape views by clicking the Swap dimensions icon . Toggle the screen resolution emulator on and off using the checkbox on the left. Select the Fit checkbox to ensures that the emulated screen remains fully visible inside your browser viewport, shrinking to fit if necessary. (This setting is simply for convenience and does not emulate the device differently.)

Customize the screen settings

To get more granular control over the screen emulator, you can tune the resolution settings directly below the device preset dropdown.

To emulate a custom screen size, manually set the CSS pixel dimensions of the device in the width and height fields.

If you want to emulate a retina device from a non-retina machine or vice versa, adjust the device pixel ratio. The device pixel ratio is the ratio between logical pixels and physical pixels. Devices with retina displays, such as the iPhone 5, have higher pixel density than standard devices, which can affect the sharpness and size of visual content.

Some examples of DPR-sensitive web components are:

  • CSS media queries such as @media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) { ... }
  • CSS image-set rules.
  • The srcset attribute on images.
  • The Javascriptwindow.devicePixelRatio property.

Note: If you have a native retina display, you'll notice that low dpi assets look pixelated while higher-dpi assets are sharp. To simulate this effect on a standard display, select the Fit checkbox and set the DPR to 2.

Save custom presets

No one wants to reconfigure the screen emulator every time they need to test a custom device. Save your custom emulations as presets so you can easily return to them later.

To save your current screen settings as a preset, open the DevTools emulation drawer by clicking the icon in the top right corner of the browser viewport.

In the device panel of the emulation drawer, click the Save as button and give your preset a name.

Now you can quickly select your screen custom emulation from the device preset dropdown.

Tip: The emulation drawer contains the same controls as the main device mode interface as well as some extras (media type and sensor controls).

Emulating Network Connectivity

Optimizing your site's performance under varying network conditions is a key aspect of developing for a mobile audience.

Device mode's network conditioning allows you to test your site on a variety of network connections, including Edge, 3G, and even offline. Select a connection from the preset dropdown to apply network throttling and latency manipulation.

Network throttling artificially limits the maximum download throughput (rate of data transfer). Latency manipulation enforces a minimum delay in connection (RTT).

Inspecting Media Queries

Since media queries are an essential part of responsive web design, device mode makes them readily accessible and easy to inspect.

To view the media query inspector, click the Media queries icon in the upper left corner of the viewport. The DevTools detect media queries in your stylesheets and display them as colored bars in the top ruler.

Media queries are color-coded as follows:

Queries targeting a maximum width.
Queries targeting widths within a range.
Queries targeting a minimum width.

Preview screen styles

Click on a media query bar to adjust the emulator resolution and preview styles for the targeted screen sizes.

View CSS

Right-click on a bar to view where the media query is defined in CSS and jump to a line of source code.

Tip: As you work with the media query inspector, you might find that you don't want to use the mobile emulator all of the time. To turn off mobile emulation without exiting device mode, click the Reset all overrides icon and refresh the page.

Preview styles for more media types

The media query inspector targets styles intended for screens. If you want to preview styles for other media types, such as print, you can do so in the media panel of the emulation drawer.

Open the DevTools emulation drawer by clicking the icon in the top right corner of the browser viewport. Then, select the media panel in the emulation drawer.

Select the CSS media checkbox and choose a media type from the dropdown list.

Emulating Device Sensors

Since most desktops don't have touch screens, GPS chips, or accelerometers, these inputs can be difficult to test on your development machine. But frequent device testing may slow down your development cycle - especially if each change requires pushing to a server and then loading the new build onto a device. Device mode reduces the overhead of testing mobile features by emulating common mobile device sensors.

To access the sensor controls, open the DevTools emulation drawer by clicking the icon in the top right corner of the browser viewport. Then, select the sensors panel in the emulation drawer.

Note: If your app detects sensors onload using Javascript (such as Modernizr), make sure that you reload the page after enabling sensor emulators.

Trigger touch events

The touch screen emulator lets you accurately test touch events and sequences as if you were using a touch-enabled device.

Enable touch emulation by selecting the Emulate touch screen checkbox in the sensors pane of the emulation drawer.

When you interact with the emulated viewport, the cursor changes into a fingertip-sized circle and touch events (e.g. touchstart, touchmove, and touchend) fire as they would on a mobile device.

The touch emulator does not disable mouse events entirely, since mouse events can still fire on touch devices.

Devices that support touch input, such as modern Apple Macbooks, can simulate multi-touch events. For further assistance with multi-touch event simulation, see the "Developer tools" chapter of the Multi-touch web development guide on HTML5 Rocks.

Note: To trigger elem.ontouch* handlers, you must use the --touch-events command line flag. Touch emulation currently does not trigger these handlers by default.

Tip: Try out the DevTools debugger in combination with touch emulation using this fingerpaint touch screen demo.

Override geolocation data

Unlike desktops, mobile devices commonly use GPS hardware to detect location. In device mode, you can simulate geolocation coordinates to use with the Geolocation API.

Enable the geolocation emulator by selecting the Emulate geolocation coordinates checkbox in the sensors pane of the emulation drawer.

The DevTools support overriding position values for navigator.geolocation and simulating cases when geolocation data is unavailable.

Tip: Try out the geolocation emulator using this maps demo.

Simulate device orientation

If you need to test accelerometer data used with the Orientation API, you can simulate the data using the accelerometer emulator.

Enable the accelerometer emulator by selecting the Accelerometer checkbox in the sensors pane of the emulation drawer.

You can manipulate the following orientation parameters:

α
Rotation around the z-axis.
β
Left-to-right tilt.
γ
Front-to-back tilt.

You can also click on the model accelerometer and drag it to the desired orientation.

Tip: Try out the accelerometer emulator using this device orientation demo.

Limitations

Although Chrome's device mode offers many powerful emulation tools, it does have some limitations. Currently known issues are described below.

  • Device hardware
    • GPU and CPU behavior are not emulated.
  • Browser UI
    • System displays, such as the address bar, are not emulated.
    • Native displays, such as <select> elements, are not emulated as a modal list.
    • Some enhancements, such as number inputs opening a keypad, may vary from actual device behavior.
  • Browser functionality
    • WebGL operates in the emulator, but is not supported on iOS 7 devices.
    • MathML is not supported on Chrome, but is supported on iOS 7 devices.
    • iOS 5 orientation zoom bug is not emulated.
    • Line-height operates in the emulator, but is not supported on Opera Mini.
    • CSS rule limits, such as those in Internet Explorer, are not emulated.
    • Changes to window.outerWidth/window.outerHeight values based on the DPR are not emulated.
    • Variations in compositor behavior are not emulated.
  • AppCache
    • The emulator does not override the UA for AppCache manifest files or view source requests.

Despite these limitations, the device mode emulators are robust enough for most tasks. And in cases where you need to test on a real device, the remote debugging DevTools can offer additional insight.
